## 이상 없음 확인 축

- worker mods 토큰: account_id 없으면 list_accounts().first()=last_used DESC 최신 계정 ✓
- 토큰 URL 인코딩: base64/hex 토큰의 특수문자(+/=)를 %2B%2F%3D로 — TS encodeURIComponent와 실질 동일 ✓
- sha256 설치 검증 ✓, is_newer 다운그레이드 없음 ✓
- 딥링크 MODE1(무조건 overwrite=true)/MODE2(기존 토큰 보존 overwrite=false) — TS 의미 일치 ✓
- config 포맷(authPort 35565/serverStatusPort 4444 등) TS 바이트 동일 ✓
- lock 보유 중 await 없음(컴파일 Send 바운드) ✓
- 파일 감시 재시작 시 기존 watcher drop 정리 ✓

## 결론

발견 6건 — **Critical 1(M5-6)** + High 2(M5-1/M5-2) + Med 1(M5-3) 수정(`4404275`), Low 2(M5-4/M5-5) 보류(M6). M5-6은 M5 전체를 무동작으로 만든 배선 유실로, 리뷰의 최대 수확. 교훈: `never used` 경고를 마감 검증에서 흘려보내지 말 것 — 커맨드/이벤트 배선 유실의 신호다. 이후 마일스톤 마감에 `cargo build 2>&1 | grep "never used"` 0건 확인을 포함.

**M5 재검증 후 상태**: 커맨드 등록 복구로 shim→커맨드 경로 정상. 실동작 확인은 일괄 테스트(tauri dev)에서.
