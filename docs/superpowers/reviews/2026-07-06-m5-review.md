# M5 리뷰 (혜니월드 통합) — 계약/의미 축

> 커밋 범위 `5049c42`~`6c7892a`. M4b 리뷰에서 수확이 컸던 계약 불일치 + TS 의미 차이 축으로 검증.

## 발견

| # | 심각도 | 내용 | 근거 | 상태 |
|---|---|---|---|---|
| M5-1 | **High** | `workermods::find_mod_files`가 `starts_with(prefix)` 매칭 — TS는 `^{modId}-.*\.jar$`(modId 뒤 **하이픈 필수**). modId가 다른 modId의 프리픽스면(예 "hyeni" ⊂ "hyenihelper"/"hyenicore") 오검출 → ① 버전 오판 ② **install_updates의 구버전 제거에서 다른 모드 삭제**(데이터 손실급) | worker-mod-updater.ts:597 `new RegExp('^${modId}-.*\\.jar$','i')` vs workermods.rs find_mod_files | 수정 |
| M5-2 | **High (Windows)** | crash_export_report의 Downloads 경로 폴백이 `HOME`만 사용 — **Windows는 HOME 미설정**(USERPROFILE)이라 리포트 내보내기 실패. 1차 후보(`userData.parent()/Downloads`)도 잘못된 경로(`~/Library/Application Support/Downloads`)라 exists 필터로 탈락 → HOME 폴백 의존 | crash.rs downloads 결정부. Windows 주 타깃인데 실패 | 수정 |
| M5-3 | Med | apply_auth가 프로필별 config 쓰기 실패 시 `?`로 전체 중단 — TS handleAuthRequest는 프로필별 continue(부분 성공 허용) | hyeni.rs(app) apply_auth 루프 | 수정 |
| M5-4 | Low(별도) | 경로 대소문자 이원화: legacy_user_data_dir()=소문자 `hyenimc`, DB game_directory=대문자 `HyeniMC`. macOS/Windows 무해(대소문자 무시, inode 동일 확인), **Linux만 위험**. worker_mods는 렌더러 profilePath(소문자 파생) 직접 사용, 딥링크는 DB game_directory(대문자) 사용 — 표기 불일치 | M1 데이터 경로 이슈. Linux 비주력 → M6 배포 시 정리 | 보류 |
| M5-5 | Low | resourcepack/shaderpack providedByPack: 기존 Electron으로 설치된 팩은 .hyenipack-provided.json 부재 → 전부 사용자 추가로 표시. 기능 저하만(무해) | M6 폴리시 | 보류 |

## 이상 없음 확인 축

- worker mods 토큰: account_id 없으면 list_accounts().first()=last_used DESC 최신 계정 ✓
- 토큰 URL 인코딩: base64/hex 토큰의 특수문자(+/=)를 %2B%2F%3D로 — TS encodeURIComponent와 실질 동일 ✓
- sha256 설치 검증 ✓, is_newer 다운그레이드 없음 ✓
- 딥링크 MODE1(무조건 overwrite=true)/MODE2(기존 토큰 보존 overwrite=false) — TS 의미 일치 ✓
- config 포맷(authPort 35565/serverStatusPort 4444 등) TS 바이트 동일 ✓
- lock 보유 중 await 없음(컴파일 Send 바운드) ✓
- 파일 감시 재시작 시 기존 watcher drop 정리 ✓

## 결론

발견 5건 중 High 2(M5-1/M5-2)·Med 1(M5-3) 수정, Low 2 보류. 특히 M5-1은 데이터 손실, M5-2는 Windows(주 타깃) 실패라 반드시 수정.
